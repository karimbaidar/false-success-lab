# Roadmap

False Success Lab stays focused on making false-success risk concrete and
actionable for developers.

## Near Term

- Keep refund as the flagship scenario while polishing the other built-in
  scenario flows.
- Improve report-card readability for imported Markdown and JSON reports.
- Add more deterministic tests around copyable fix snippets.
- Keep the docs aligned with the recommended `false-success-lab` repo rename.

## Later

- Add more verifier-pack examples for access, CRM, infrastructure, and trading
  workflows.
- Add scenario fixtures contributed by users after privacy review.
- Add richer proof-trail filtering and receipt inspection.
- Improve backend deployment notes for public GitHub scanning.

## Non-Goals

- Do not claim static scans prove production safety.
- Do not ask browsers to inspect arbitrary local folders.
- Do not duplicate core scanner or receipt logic from `agent-consistency`.
